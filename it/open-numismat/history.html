Cosa c&#x27;è di nuovo in OpenNumismat 1.4 (31 Gennaio 2013)

  • Implementata la creazione di resoconti (basato su modelli Jinja2)
  • Implementata l'esportazione della lista personalizzata (Resoconto> Salva lista corrente...)
  • Aggiunta la possibilità di inserire tutti i dati in campi numerici (enable Free format numeric fields in the Settings dialog) ( issue 6 ,  issue 10 )
  • Corretto controllo aggiornamenti automatico (con frequenti avvii dell'applicazione non funzionava bene)
  • Aggiunto comando Compressione (menu Collezione) che libera lo spazio su disco riservato ai motori database
  • Cambiato il nome del file di default quando si salvano le immagini di monete in un file
  • Corretto un possibile errore aprendo il file specificato nel campo URL
  • Corretto un possibile errore quando si crea una nuova collezione
  • Aggiornato il menu delle applicazioni - aggiunti alcuni elementi di uso comune

Cosa c&#x27;è di nuovo in OpenNumismat 1.3.1 (05 Dicembre 2012)

  • Corretto risparmio larghezza delle colonne della tabella.
  • Aggiunto un pulsante per l'apertura URL (sezione Classificazione) e inserito un link per aprire file su disco.
  • Corretta la scelta del valore nel riferimento quando si modifica la moneta.
  • Corretta la formattazione nei campi descrizione e testo monete (Disegno, Soggetto, etc).
  • Il file della collezione può essere passato come argomento della riga di comando.
  • Corretto possibile bug durante la modifica delle commissioni negative ( issue 9 ).
  • Aggiunto eliminare i file temporanei creati durante l'apertura delle immagini.
  • Altri minori cambiamenti.

Cosa c&#x27;è di nuovo in OpenNumismat 1.3 (23 Novembre 2012)

  • Riscrittura del database per migliorare le prestazioni (la conversione dei file della collezione avviene automaticamente quando li si apre in una nuova versione di OpenNumismat).
  • Aggiunti nuovi campi: Quantità, URL, Codice a barre.
  • Aggiunto controllo automatico degli aggiornamenti (passata nella finestra di dialogo Impostazioni).
  • Implementata l'importazione da Collection Studio 3.65. Per importare, è necessario prima esportare la collezione esistente di una tabella in formato XML utilizzando il menu Collection Studio: Strumenti> Espota....
  • Aggiunta la possibilità di proteggere i dati sulla collezione con una password (Nota: la crittografia dei dati non viene eseguita, in modo che la password di protezione non può essere considerata completa.) L'impostazione della password è nel menuCollezione> Imposta password....
  • Migliorata l'importazione da CoinManage:
    • supporto per i nuovi campi Quantità e Codici a barre;
    • aggiunta alle monete da collezione di immagini relative alle descrizioni (elenco varietà);
    • prezzi d'importazione.
  • Aggiunto pulsante Ripristina predefinite in Impostazioni per ripristinare i nomi di tutti i campi moneta ai suoi valori originali.
  • Corretto calcolo delle commissioni negative per modificare le informazioni per l'acquisto / vendita delle monete.
  • Corretto rilevamento della lingua corrente nella finestra delle Impostazioni.
  • Corretto riempimento dei riferimenti, a seconda del paese di raccolta dati.
  • Tasto Esc nella finestra di modifica moneta chiude la finestra.
  • Doppio click sulla barra delle schede crea una nuova lista.
  • Aggiunta la visualizzazione di una barra di avanzamento durante la rimozione delle monete.
  • Migliorata traduzione in inglese.
  • Migliorato l'aspetto delle Impostazioni .
